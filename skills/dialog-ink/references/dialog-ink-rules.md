# dialog-ink preflight rules

**Loaded by `--check` flag.** Apply ALL checks before writing.

## Context verification

Before writing any scene, verify you have:

1. **Characters present** — Who is in the scene? Who enters/exits during?
2. **Location** — Where does this happen? What's the physical space?
3. **Relationships** — What is the dynamic between these characters right now?
4. **Stakes** — What does each character want from this exchange?
5. **Prior context** — What just happened? What do the characters know?

If any of these is missing: signal it to the user, ask for the missing element.
Do not invent context — the author provides it.

## Pre-writing checklist

Before delivering the scene, verify:

- [ ] Scene opens with space and physical setup
- [ ] Every character who speaks has a physical presence (position, movement)
- [ ] Dialogue sounds spoken, not written — oral register
- [ ] No line of dialogue exists without body/action context
- [ ] Entrances and exits are staged (not teleported)
- [ ] Social rituals present where expected (greetings, reactions to arrivals)
- [ ] Silences and pauses are noted as stage directions
- [ ] No interior monologue or narrator voice leaked in
- [ ] No literary metaphors or atmospheric description
- [ ] Monologues broken by action or interruption (max 4 lines unbroken)
- [ ] Subtext exists — something is communicated through action, not words
